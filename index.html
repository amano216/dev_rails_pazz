🚀 環境構築チュートリアル

Dockerを使ったモダンなRails開発環境

このガイドでは、Dockerを使って、クリーンで再現性の高いRails開発環境を立ち上げる典型的な手順を解説します。各コマンドをクリックすると、ページ下部の詳細なリファレンスにジャンプします。

✅ 前提条件

このフローを始める前に、プロジェクトのルートディレクトリにDockerfileとdocker-compose.ymlが用意されている必要があります。

  1. 環境の構築: 設計図（Dockerfile）を元に、Railsアプリ専用の環境（イメージ）を構築します。
    docker compose build
  2. コンテナの起動: 構築したイメージを元にコンテナ（仮想PC）をバックグラウンドで起動します。
    docker compose up -d
  3. データベースの準備: コンテナの中に入り、データベースの作成と初期データの投入を行います。
    docker compose exec web bash → rails db:create → rails db:migrate
  4. 開発サーバーの起動: フロントエンドとバックエンド、2つのターミナルでそれぞれサーバーを起動します。
    (ターミナル1) docker compose exec web bash → yarn build --watch
    (ターミナル2) docker compose exec web bash → bin/rails s -b 0.0.0.0
  5. アプリケーションの確認: ブラウザで http://localhost:3000 にアクセスします。
  6. 開発の終了: 各サーバーをCtrl+Cで停止し、コンテナを完全に終了させます。
    docker compose down

🌱 新機能開発チュートリアル

一般的な機能開発の基本的な流れ

新しい機能を追加する際の、典型的なコマンドの流れをまとめました。各コマンドをクリックすると、詳細なリファレンスにジャンプします。

  1. 新しい作業用のブランチを作成します: git checkout -b [ブランチ名]
  2. 必要なモデルを生成します: rails g model [モデル名] [カラム...]
  3. 必要なコントローラーを生成します: rails g controller [名前] [アクション...]
  4. データベースの構造を更新します: rails db:migrate
  5. (エディタでコードを編集します)
  6. 変更をステージングします: git add .
  7. 変更をコミットします: git commit -m "メッセージ"
  8. リモートリポジトリに変更を送信します: git push origin [ブランチ名]
  9. GitHubでプルリクエストを作成します: gh pr create

🐛 デバッグチュートリアル

バグや問題を調査するときの基本的な流れ

何か問題が起きた際の、原因調査のための典型的なコマンドの流れです。各コマンドをクリックすると、詳細なリファレンスにジャンプします。

  1. 何が起きているか、ログの最後の部分を確認します: tail -f log/development.log
  2. 最近の変更履歴を確認し、原因のあたりをつけます: git log --oneline
  3. 特定のファイルの変更差分を確認します: git diff [ファイル名]
  4. 怪しい箇所にデバッガを仕込み、実行を一時停止します: binding.pry
  5. Railsコンソールを起動し、データを直接確認・操作します: rails console
  6. 実行中のプロセスを確認し、不要なプロセスを停止します: ps aux → kill [プロセスID]

🗄️ データベースマイグレーション管理

モデルの追加からデータの投入まで

データベースの構造を変更し、データを管理する際の基本的なマイグレーションフローです。

  1. モデルとマイグレーションファイルの生成: 新しいテーブルの設計図を作成します。
    rails g model Post title:string content:text
  2. データベースへの反映: 設計図を元に、実際のデータベースにテーブルを作成します。
    rails db:migrate
  3. 適用状態の確認: どのマイグレーションが適用済みかを確認します。
    rails db:migrate:status
  4. 一つ前の状態に戻す: 直前のマイグレーションを取り消します。
    rails db:rollback
  5. カラム追加のマイグレーション生成: 既存のテーブルに新しいカラムを追加します。
    rails g migration AddAuthorToPosts author:string
  6. 再度データベースへ反映: 新しいカラムをデータベースに追加します。
    rails db:migrate
  7. スキーマからのDB再構築: チームメンバーが追加したマイグレーションをまとめて適用します。
    rails db:schema:load
  8. 初期データの投入: 開発に必要な初期データを投入します。
    rails db:seed

🤝 Git合わせ技チュートリアル

よく使うGitコマンドの組み合わせ

実際の開発シーンで頻繁に登場する、複数のGitコマンドを組み合わせたワークフローを紹介します。

最新のmainブランチを取り込んでから開発を始める

  1. メインブランチに切り替えます: git checkout main
  2. リモートの最新の状態を取得します: git pull
  3. 新しい機能開発用のブランチを作成します: git checkout -b new-feature

作業の区切りでコミットし、リモートに保存する

  1. 変更の状態を確認します: git status
  2. 全ての変更をステージングします: git add .
  3. 変更内容をコミットします: git commit -m "キリの良い作業完了"
  4. リモートリポジトリに変更を保存します: git push origin new-feature

mainブランチの変更を作業ブランチに取り込む

  1. リモートの最新情報を取得します: git fetch origin
  2. 作業ブランチの起点を最新のmainに移動させます: git rebase origin/main

ローカルでの変更を一旦退避して、別の作業を優先する

  1. 現在の変更を一時的に退避させます: git stash
  2. (別のブランチで緊急の作業など...)
  3. 退避させた変更を元に戻します: git stash pop

直前のコミットに修正を加える

  1. (ファイルを追加・修正します)
  2. 修正内容をステージングします: git add .
  3. 直前のコミットに含めて修正します: git commit --amend --no-edit
  4. 修正したコミットを強制的にプッシュします: git push -f origin new-feature
