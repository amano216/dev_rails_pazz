🚨 よくあるエラーパターンと解決法

NoMethodError

undefined method 'name' for nil:NilClass

nilオブジェクトにメソッドを呼び出し。&.演算子やif文で対処

NameError

uninitialized constant User

クラスが定義されていない。ファイル名とクラス名の確認

SyntaxError

unexpected end-of-input

endの不足やカッコの不一致。インデントを確認

ActiveRecord::RecordNotFound

Couldn't find User with 'id'=1

存在しないレコード。find_byやrescueで対処

ActiveRecord::RecordInvalid

Validation failed: Email can't be blank

バリデーションエラー。save!の代わりにsaveを使用

ActionController::ParameterMissing

param is missing or empty: user

必須パラメータ不足。Strong Parametersを確認

ActionView::MissingTemplate

Missing template users/index

ビューファイルが存在しない。ファイルパスを確認

LoadError

cannot load such file -- bcrypt

Gemが見つからない。bundle installを実行

PG::ConnectionBad

could not connect to server

PostgreSQLが起動していない。brew services start postgresql

Mysql2::Error::ConnectionError

Can't connect to MySQL server

MySQLが起動していない。sudo service mysql start

ActiveRecord::PendingMigrationError

Migrations are pending

未実行のマイグレーション。rails db:migrateを実行

ActiveRecord::StatementInvalid

SQLite3::SQLException: no such column

カラムが存在しない。マイグレーションファイルを確認

ActionController::InvalidAuthenticityToken

Can't verify CSRF token

CSRFトークンエラー。フォームにcsrf_meta_tagsを追加

Net::ReadTimeout

execution expired

タイムアウト。タイムアウト時間を延長またはリトライ処理

ActiveRecord::DuplicateEntry

Duplicate entry for key 'index_users_on_email'

一意制約違反。データの重複を確認

ArgumentError

wrong number of arguments (given 2, expected 1)

引数の数が違う。メソッド定義を確認

TypeError

no implicit conversion of Symbol into Integer

型の不一致。配列とハッシュの混同を確認

Bundler::GemNotFound

Could not find gem 'rails'

Gemが見つからない。bundle installを実行

Webpacker::Manifest::MissingEntryError

Can't find application.js in manifest.json

Webpackerコンパイルエラー。rails webpacker:compileを実行

ActiveStorage::FileNotFoundError

Could not find file

アップロードファイルが見つからない。ストレージを確認

Redis::CannotConnectError

Error connecting to Redis

Redisが起動していない。redis-serverを起動

Errno::EADDRINUSE

Address already in use - bind(2) port 3000

ポートが使用中。lsof -i:3000でプロセスを確認してkill

ActiveJob::DeserializationError

Error while trying to deserialize arguments

ジョブの引数エラー。GlobalIDを使用またはIDを渡す

StandardError - カスタムエラー処理

rescue StandardError => e

汎用的なエラーキャッチ。ログ出力して適切に処理

🌐 HTTPメソッド・コマンド

GET - データ取得

curl -X GET https://api.example.com/users

サーバーからデータを取得。リソースの読み取り専用

POST - データ作成

curl -X POST -d '{"name":"John"}' https://api.example.com/users

新しいリソースを作成。サーバーにデータを送信

PUT - データ更新（全体）

curl -X PUT -d '{"name":"Jane"}' https://api.example.com/users/1

既存リソース全体を置き換え。完全な更新

PATCH - データ更新（部分）

curl -X PATCH -d '{"email":"new@example.com"}' https://api.example.com/users/1

既存リソースの一部を更新。部分的な変更

DELETE - データ削除

curl -X DELETE https://api.example.com/users/1

指定したリソースを削除。サーバーから完全に削除

HEAD - ヘッダー取得

curl -I https://api.example.com/users

レスポンスヘッダーのみ取得。ボディなし

OPTIONS - 許可メソッド確認

curl -X OPTIONS https://api.example.com/users

サーバーがサポートするHTTPメソッドを確認

curl - 基本的なGETリクエスト

curl https://api.example.com/data

最もシンプルなHTTPリクエスト実行

curl - ヘッダー付きリクエスト

curl -H "Authorization: Bearer token" https://api.example.com

認証トークンなどのヘッダーを追加

curl - JSONデータ送信

curl -H "Content-Type: application/json" -d '{"key":"value"}' https://api.example.com

JSON形式でデータを送信

curl - フォームデータ送信

curl -X POST -F "name=value" -F "file=@path/to/file" https://api.example.com

フォーム形式でデータとファイルを送信

curl - Basic認証

curl -u username:password https://api.example.com

Basic認証でアクセス

curl - クッキー送信

curl -b "session=abc123" https://api.example.com

クッキーを含めてリクエスト

curl - クッキー保存

curl -c cookies.txt https://api.example.com/login

レスポンスのクッキーをファイルに保存

curl - タイムアウト設定

curl --connect-timeout 10 --max-time 30 https://api.example.com

接続と全体のタイムアウトを設定

curl - リダイレクト追跡

curl -L https://api.example.com

リダイレクトを自動的に追跡

curl - プロキシ経由

curl -x http://proxy:8080 https://api.example.com

プロキシサーバー経由でアクセス

curl - SSL証明書検証スキップ

curl -k https://api.example.com

開発環境でSSL証明書の検証をスキップ

curl - レスポンスを整形

curl https://api.example.com/data | jq

JSONレスポンスを見やすく整形

curl - ファイルダウンロード

curl -O https://example.com/file.pdf

ファイルをダウンロードして保存

curl - 詳細情報表示

curl -v https://api.example.com

リクエスト/レスポンスの詳細を表示

curl - 静かモード

curl -s https://api.example.com

進捗バーなどを非表示にして実行

HTTPie - GET

http GET api.example.com/users

HTTPieを使った見やすいGETリクエスト

HTTPie - POST

http POST api.example.com/users name="John" age=30

HTTPieでJSON形式のPOSTリクエスト

wget - ファイル取得

wget https://example.com/file.zip

ファイルをダウンロード

wget - 再帰的ダウンロード

wget -r https://example.com/docs/

ディレクトリ全体を再帰的にダウンロード

Rails - HTTPリクエスト（Net::HTTP）

Net::HTTP.get(URI('https://api.example.com/data'))

RubyのNet::HTTPライブラリでGETリクエスト

Rails - HTTParty gem

HTTParty.get('https://api.example.com/users')

HTTParty gemを使った簡単なHTTPリクエスト

Rails - Faraday gem

Faraday.get('https://api.example.com/data')

Faraday gemでHTTPリクエスト

Rails - RestClient gem

RestClient.get 'https://api.example.com/users'

RestClient gemでシンプルなAPI呼び出し

Rails - API認証付きリクエスト

HTTParty.get(url, headers: { 'Authorization' => "Bearer #{token}" })

認証トークン付きのAPIリクエスト

ステータスコード - 200 OK

HTTP/1.1 200 OK

リクエスト成功。正常にデータを返す

ステータスコード - 201 Created

HTTP/1.1 201 Created

新しいリソースの作成成功

ステータスコード - 204 No Content

HTTP/1.1 204 No Content

成功したが返すコンテンツなし

ステータスコード - 400 Bad Request

HTTP/1.1 400 Bad Request

不正なリクエスト。パラメータエラー

ステータスコード - 401 Unauthorized

HTTP/1.1 401 Unauthorized

認証が必要。認証情報なしまたは無効

ステータスコード - 403 Forbidden

HTTP/1.1 403 Forbidden

アクセス禁止。権限不足

ステータスコード - 404 Not Found

HTTP/1.1 404 Not Found

リソースが見つからない

ステータスコード - 422 Unprocessable Entity

HTTP/1.1 422 Unprocessable Entity

検証エラー。データは正しいが処理できない

ステータスコード - 500 Internal Server Error

HTTP/1.1 500 Internal Server Error

サーバー内部エラー

ステータスコード - 502 Bad Gateway

HTTP/1.1 502 Bad Gateway

ゲートウェイまたはプロキシのエラー

ステータスコード - 503 Service Unavailable

HTTP/1.1 503 Service Unavailable

サービス利用不可。メンテナンス中など
